If you are experiencing a technical problem with Session Buddy, try the following:
1Restart Chrome
Make sure to shut down Chrome completely.
  • On Windows: after closing all Chrome windows, check the tool tray for the Chrome icon. If it is present, right-click it and click Exit.
  • On OS X: shut down Chrome by pressing Command+Q. Check the Dock for the Chrome icon. If present, right-click it and click Quit if this menu item is available.

If the problem persists after starting Chrome again, try step 2.
2Reinstall Session Buddy
Make sure to back up your Session Buddy data before reinstalling so that you can restore it into the new installation. See:

If you are able to access Session Buddy, see: How to Back Up and Restore Session Buddy Data.

If you are NOT able to access Session Buddy, check out the Advanced Backup and Restore Instructions.

Once you've backed up your data, uninstall Session Buddy and completely shut down and restart Chrome (see step 1 above).

After restarting Chrome, reinstall Session Buddy from the Chrome Web Store.

If you are then able to open Session Buddy without issue, proceed to restore your data according to the instructions linked above corresponding to the type of backup you performed.

If this step does not solve the problem, proceed to step 3.
3Contact support@sessionbuddy.com
Click here to compose a support email message

Alternatively, you can click here to copy diagnostic details to the clipboard and then paste this into an email to support@sessionbuddy.com.

When contacting support, please provide as much detail as possible about the trouble you're having. The more information you provide, the better equipped we are to resolve the problem.

Some helpful information to include:
  1. Specifically what steps lead to the observed failure?
  2. What are you expecting to happen and what is happening instead?
  3. Can the observed failure be reliably reproduced?
  4. When did the trouble start? (eg, after a recent update? after a recent crash?)
  5. What error messages are displayed by Session Buddy?
  6. What error messages are displayed in the console? (see the following linked instructions for accessing Session Buddy's console)
  7. What steps have you taken in an effort to correct the problem?
  8. Are you synching Chrome extensions or bookmarks through Google?

Additionally helpful items to attach:
  1. A screenshot or screencast demonstrating the problem. This should include views of the console and Session Buddy's settings dialog, if possible.
  2. The Session Buddy local user data folder, where sessions and user settings are stored. You do not need to submit this when you first contact support, but we may ask for it to help us troubleshoot if the problem is suspected to be data-related.
